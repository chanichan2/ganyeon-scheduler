# 손세상 갠연 스케줄러

관리자가 **부원별 가용시간**과 이미 확정된 **팀연습**을 한 화면에서 보면서,
1시간 단위 칸을 클릭해 **갠연(개인연습)** 을 잡고, 결과를 TSV 로 내보내
sonsesang2026 스프레드시트의 연습일정 탭에 붙여넣는 도구입니다.

예약 상태는 Apps Script 서버(PropertiesService)에 저장되어 어느 기기에서
열어도 같은 상태가 보입니다.

- 가용시간·조사 기간: 기존 **sonsesangscheduler**(일정조사) Apps Script 에서 읽기만 합니다.
- 팀연습·갠연 예약: 이 프로젝트 전용으로 **새로 만드는 스프레드시트 + 새 Apps Script** 가 담당합니다.

> ⚠️ 기존 sonsesangscheduler Apps Script 배포·코드, sonsesang2026 라이브
> 사이트는 절대 수정하지 마세요. 이 프로젝트는 완전히 별도의 새 배포입니다.

---

## 배포 절차 (처음 한 번만, 순서대로)

> **순서가 중요해요.** 반드시 **① 서버(Apps Script)를 먼저 배포**하고,
> **② 사이트(Cloudflare Pages)를 나중에** 배포합니다.
> 사이트는 .env 에 적힌 서버 주소가 있어야 동작하기 때문입니다.

### 1단계 — 새 스프레드시트 만들기

1. [sheets.google.com](https://sheets.google.com) 에서 **새 스프레드시트**를 만듭니다.
   (기존 일정조사/연습일정 스프레드시트가 아니라 **새 파일**입니다!)
2. 아래쪽 탭 이름을 더블클릭해 `연습일정` 으로 바꿉니다.
3. 1행에 헤더를 입력합니다 (왼쪽부터 순서대로):

   | A | B | C | D | E | F |
   |---|---|---|---|---|---|
   | 날짜 | 시작 | 종료 | 곡명 | 연습실 | 참여부원 |

4. sonsesangscheduler 에서 내보낸 **팀연습 TSV** 를 A2 셀에 붙여넣습니다.
   (연습실 열은 비어 있어도 됩니다. 나중에 팀연습이 바뀌면 2행 이하를 지우고
   다시 붙여넣으면 사이트에 자동 반영됩니다.)

### 2단계 — Apps Script 설치·배포 (서버 먼저!)

1. 방금 만든 스프레드시트에서 **확장 프로그램 → Apps Script** 를 엽니다.
2. 편집기에 있는 기본 코드를 전부 지우고, 이 레포의
   **`apps_script_ganyeon.gs`** 파일 내용 전체를 복사해 붙여넣습니다.
3. **저장** (Ctrl+S 또는 ⌘+S).
4. **스크립트 속성에 관리자 비밀번호 설정:**
   - 왼쪽 톱니바퀴(⚙ 프로젝트 설정) → 아래로 스크롤 → **스크립트 속성** →
     **스크립트 속성 추가**
   - 속성: `ADMIN_TOKEN` / 값: 원하는 관리자 비밀번호 → 저장
   - 이 비밀번호를 아는 사람만 예약을 바꿀 수 있어요.
     설정하지 않으면 모든 변경이 거부됩니다.
5. **웹앱으로 배포:**
   - 오른쪽 위 **배포 → 새 배포**
   - 유형 선택(⚙) → **웹 앱**
   - "액세스 권한이 있는 사용자" → **모든 사용자**
   - **배포** 클릭 → 권한 승인(내 계정 선택 → 고급 → 이동 → 허용)
6. 발급된 **웹 앱 URL** (https://script.google.com/macros/s/…/exec) 을
   복사해 둡니다. → 이것이 `VITE_GANYEON_API_URL` 입니다.

> 나중에 Apps Script 코드를 수정하면: **배포 → 배포 관리 → (연필 아이콘) →
> 버전: 새 버전 → 배포**. URL 은 그대로 유지됩니다.

### 3단계 — .env 값 넣기

1. 레포의 `.env.example` 을 복사해 `.env` 파일을 만듭니다.
2. 두 값을 채웁니다:

   ```
   VITE_SURVEY_API_URL=  ← 기존 일정조사(sonsesangscheduler) Apps Script 웹앱 URL
   VITE_GANYEON_API_URL= ← 2단계에서 복사한 새 웹앱 URL
   ```

### 4단계 — Cloudflare Pages 연결 (사이트는 마지막!)

1. 이 레포를 GitHub 에 올립니다 (`.env` 는 자동으로 제외됩니다).
2. [Cloudflare Pages](https://pages.cloudflare.com) → **Create a project** →
   GitHub 레포 연결.
3. 빌드 설정:
   - Framework preset: **Vite**
   - Build command: `npm run build`
   - Build output directory: `dist`
4. **환경 변수(Environment variables)** 에 3단계의 두 값을 추가합니다:
   - `VITE_SURVEY_API_URL`
   - `VITE_GANYEON_API_URL`
5. **Save and Deploy** → 발급된 `*.pages.dev` 주소로 접속해 확인합니다.

## 사용법

- **열람**: 누구나 접속해서 부원표를 볼 수 있어요.
  - 흰색 = 가능 시간, 어두운 색 = 불가능, 색 막대 = 팀연습, 인디고 = 갠연 예약.
  - 부원 이름 옆 배지 `미정` = 그 날짜 가용시간이 미정.
  - 이름 아래 파란 숫자 = 전체 기간 누적 갠연 시간 (유효 구간의 실제 분 합계).
  - 넓은 화면(아이패드 가로/PC, 1024px 이상)에서는 전체 시간축이 가로 스크롤
    없이 한 화면에 들어오고, 주간 스트립이 2주(14일)로 확장돼요.
    PC 에서는 ←/→ 키로 이전/다음 날짜로 이동할 수 있어요.
- **관리자 모드**: 오른쪽 위 **관리자** 버튼 → 비밀번호(= `ADMIN_TOKEN` 값) 입력.
  - 칸 클릭 = 예약/취소 토글. 실제로 잡히는 시간은
    "그 칸 ∩ 가용시간 − 팀연습" 으로 자동 계산돼요.
  - 유효 시간 합계가 30분 미만인 칸은 클릭되지 않아요.
- **TSV 내보내기** (관리자 전용): **TSV 내보내기** 버튼 → 미리보기 모달에서
  실제 복사될 행을 확인한 뒤 **복사** → sonsesang2026 스프레드시트의 연습일정
  탭 **2행 이하**에 붙여넣기. (헤더 행은 포함되지 않아요.)
  - 행 병합은 **실제 겹침** 기준: 어떤 부원의 유효 구간도 가로지르지 않는
    정각 경계(예: 오전 갠연 10~13 과 오후 갠연 13~15 가 13시에 맞닿기만 한 경우)
    에서는 자동으로 행이 나뉘어요.
  - 모달의 시간축 스위치(사슬 = 연결, 가위 = 절단)를 탭해 경계를 수동으로
    바꿀 수 있어요. 수동 설정은 서버에 저장되어 기기 간 동기화되고, 인디고 점으로
    표시되며, **자동값으로 초기화** 버튼으로 날짜 단위 삭제할 수 있어요.
- **경고 패널**: 화면 상단에 노란 패널이 보이면 꼭 펼쳐서 확인하세요.
  파싱 실패/이름 불일치/기간 밖 날짜/유효 구간이 사라진 예약 등이 모두 나열됩니다.
  해석에 실패한 입력은 절대 추측하지 않고 "불가능" 으로 안전 처리됩니다.

### 데이터가 바뀌었을 때

- **팀연습이 바뀌면**: 새 스프레드시트 연습일정 탭의 2행 이하를 지우고 새 TSV 를
  붙여넣은 뒤, 사이트에서 새로고침(↻)만 하면 됩니다. 기존 갠연 예약의 유효
  구간은 자동으로 다시 계산돼요.
- 팀연습/가용시간 변경으로 유효 구간이 30분 미만이 된 예약은 조용히 사라지지
  않고 **회색 빗금**으로 남으며 경고 패널에 표시됩니다. 관리자가 그 칸을 클릭해
  직접 제거할 수 있어요.
- 연습일정 탭에 곡명이 정확히 `갠연` 인 행이 있으면 무시되고 경고에 표시됩니다
  (예약의 진실은 서버에만 있어서 이중 카운트를 막기 위함).

## 개발

```bash
npm install        # 의존성 설치
npm run dev        # 개발 서버 (http://localhost:5173)
npm test           # vitest 전체 실행
npm run build      # tsc -b && vite build → dist/
```

### 구조

| 파일 | 역할 |
|---|---|
| `src/availability.ts` | 가용시간 셀 파서 (`normalizeRaw`, `parseAvailability` — sonsesangscheduler 포팅) |
| `src/ranges.ts` | 분 단위 정수 구간 헬퍼 (`mergeRanges`, `subtractRange`, `fmtHM`) |
| `src/members.ts` | 참여부원 셀 토큰화 (sonsesang2026 `parseWorkbook` 에서 추출) |
| `src/memberWindow.ts` | 괄호 시간 메모 파서 (sonsesang2026 재사용) |
| `src/dates.ts` | 날짜 정규화/연도 추론 (해 넘김 포함) |
| `src/teamSchedule.ts` | 연습일정 탭 행 파싱 + 경고 |
| `src/effective.ts` | 클리핑 엔진 — 칸 ∩ 가용 − 팀연습, 30분 룰 |
| `src/export.ts` | TSV 내보내기 (run 병합, `exportMemberEntry`) |
| `src/model.ts` | 화면 모델 조립 + booking 대조/누적시간 |
| `src/useBookings.ts` | 낙관적 업데이트 + 직렬 op 큐 + 롤백 |
| `src/useAdmin.ts` | 관리자 토큰 (localStorage + verify POST) |
| `apps_script_ganyeon.gs` | 새 스프레드시트용 Apps Script 백업본 |

### 설계 원칙

- 모든 시간 계산은 **분 단위 정수** — 표시 단계에서만 포맷.
- 파싱 실패 데이터는 **추정 금지** — 불가능 처리 + 경고 패널.
- 예약의 진실은 **서버(PropertiesService)** 뿐 — localStorage 는 관리자
  토큰 저장에만 사용.
- 변경 액션의 토큰 검증은 **반드시 서버**에서 (`ADMIN_TOKEN` 스크립트 속성).
- 예약은 클릭한 1시간 칸 단위로 저장하고, 유효 구간은 렌더링/내보내기
  시점에 항상 재계산 — 팀연습 재붙여넣기/가용시간 수정이 자동 반영.
- 가로 스크롤은 단일 스크롤 컨테이너 — 중첩 overflow 금지.
